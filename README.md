# MCU Real-Time Anomaly Detection

Real-time bearing anomaly detection on STM32F4 using TensorFlow Lite for Microcontrollers (TFLM).

This project trains an autoencoder on CWRU vibration data, quantizes it to full int8 LiteRT/TFLite, converts model artifacts to C headers, and runs anomaly detection on a STM32F407 Discovery board.

## Purpose

- Train a compact autoencoder from vibration features.
- Deploy a quantized int8 model to MCU with TFLM.
- Detect anomalies online from a 48 kHz sensor stream.
- Trigger a simple hardware output (LED) from reconstruction error thresholding.

## High-Level Pipeline

1. Download and format CWRU dataset.
2. Extract time-domain features.
3. Train autoencoder and compute anomaly threshold.
4. Convert to fully quantized int8 `.tflite` model.
5. Export model, scaler, threshold, and sensor source data to C headers/sources.
6. Build and flash firmware with PlatformIO.

## Repository Layout

- `python/`: data prep, training, conversion, evaluation, and test scripts.
- `scripts/`: shell wrappers for full training/conversion/build steps.
- `model/`: trained and quantized model artifacts.
- `include/`: generated C headers (`scaler.h`, `error_threshold.h`, model header).
- `lib/`: firmware libraries (`model_interpreter`, `features_extraction`, `sensor_source`, static TFLM lib).
- `src/main.cpp`: MCU application loop.
- `datasets/`: raw/formatted data and split features.
- `tflite-micro/`: vendored TFLM source tree.

## Requirements

- Linux
- Python 3.12+ recommended
- PlatformIO CLI (`pio`)
- STM32F407 Discovery board (or compatible target configured in `platformio.ini`)

## Setup

Create and populate the Python environment:

```bash
./scripts/build_env.sh
source mcu_anomaly_env/bin/activate
```

Build the static TFLM library used by PlatformIO:

```bash
 git submodule init
 git submodule update
./scripts/build_litert.sh
```

## How To Use

The repository already contains the pretrained model. If however you want to train the model and export
it to LiteRT format by yourself, run the following commands : 
### 1) Train and Prepare ML Artifacts

Run the full ML pipeline:

```bash
./scripts/build_model.sh
```

This pipeline does:

- dataset download/formatting
- feature extraction
- model training
- full int8 conversion (`python/convert_to_litert.py`)
- model evaluation
- model header export (`scripts/convert_model_to_carray.sh`)

Export scaler and threshold headers (if needed separately):

```bash
./scripts/convert_scaler_to_carray.sh
./scripts/convert_error_threshold_to_c.sh
```

Generate sensor source C files from dataset signals:

```bash
source mcu_anomaly_env/bin/activate
python python/convert_sensor_source_to_c.py
```

### 2) Build and Flash Firmware

```bash
pio run
pio run --target upload
```

Debug:

```bash
pio debug
```

## Benchmark

### Current Build Footprint (Measured)

Measured with `pio run` on `disco_f407vg` (STM32F407VGT6):

- Flash: 70,600 bytes / 1,048,576 bytes (6.7%) + 375,200 bytes to simulate sensor data
- RAM: 9,616 bytes / 131,072 bytes (7.3%)

### Model Runtime Characteristics

- Input window size: 1024 samples (`kSensorRawSampleCount`)
- Sampling rate: 48,000 Hz (`kSensorSampleRateHz`)
- Tensor arena size: 3000 bytes (`kTensorArenaSize`)
- Features per inference: 9 (`kScalerFeatureCount`)
